Snapshot

Problem

There is no shortage of popular choices when it comes to cloud storage. Cloud storage is safe way to back up files, save things for later and to collaborate with others. However, what happens when a user saves a file to one cloud storage application and then needs to transfer it to another?

While each individual cloud storage service may provide a good user experience, the experience starts to suffer when working across services. People tend to have multiple cloud accounts and working across them can be cumbersome and require multiple steps creating workflow friction.

Solution

Focal eases this issue by providing an simple way to manage multiple cloud storage services from a central application. The user has the ability to search files and manage files across services simplifying their workflow. Focal also implements a user friendly user interface that is similar to many cloud storage applications to ease the user learning process and increase adoption rates.

User Research

To better understand the how users incorporate cloud storage into their digital lives I started my research by conducting a user survey sending it to public forums, my social media accounts and email contacts. I was looking to understand the context of how they use cloud storage and their pain points and motivations.

Survey Insights

01

Most people use more than one cloud storage application

84% of people used more than one cloud storage application. This indicates that people are managing several accounts and that there may be an opportunity to simplify the experience between disparate storage applications.

02

People use cloud storage in two different contexts.

84% of the respondents indicated that they used cloud storage for personal or family organization. 68% of the those people also indicated that they use it for work. This indicated to me that people often have two different workflows that can further complicate how they use cloud storage.

User Pain Points

Logging in from different devices

Unsure of file location

Remembering passwords

Figuring out how to use it

It became apparent that users were having issues with organization and navigating the way they use cloud storage. I wanted to be careful in not just offering yet another cloud storage service that would introduce further complexity, but one that could simplify the way users manage their current services.

Competative Analysis

I also wanted to better understand the current landscape of cloud storage and how other services positioned themselves in the market place. I started by analyzing the most popular cloud storage applications that were indicated in the user survey: Google Drive, Dropbox and Evernote. This also gave me more insight to the functionality and user flows that people would be familiar with. This would help inform the development of the user flows, UX, UI elements development later in the process.

User Stories

The user stories were created to identify key features users would want to have and then prioritize them in order to design the MVP and stay within scope.

I want to be able to register for an account.

I want to be able to a sign-in to my account.

I want to be able to link to my existing external cloud storage applications.

I want to be able to save and upload files to my account from my device.

I want to be able to organize my files into folders.

I want to be able to share and transfer files and folders to others.

I want to be able to collaborate on documents with others.

I want to be able to access my files from any device.

I want to be able to delete files.

I want to be able to unlink and edit my external accounts.

I want to be able to create a document.

I want an onboarding tutorial.

Prototyping

Low-Fidelity

I created several wireframe sketches based off of the user stories. Once satisfied with the general layouts, I created low-fidelity wireframes in sketch, which I then brought into InVision to create a low-fidelity prototype.

User Testing

To ensure that users are able to complete the highest priority user stories I conducted a usability test and recorded feedback and confusing for future iterations.

One thing that come to light during the user test was the desire to make less of a distinction between linked storage accounts and files saved directly to Focal. As one tester put it, “All storage is equal to me.”

High-Fidelity

After garnering feed back on the low-fidelity prototype, I incorporated the branding elements to the wireframes to produce the high-fidelity prototype using sketch, Craft plugin and Invision.

I then conducted another usability test to get more specific feedback on the application of the design elements, UI, any accessibility issues the may impact experience.

Initial prototype with bottom menu.

Further feed back suggested that the drop down task menu introduced too much complexity of choice. I simplified this screen by removing the bottom navigation which is now incorporated withing the main nav menu.

Final Thoughts

People are very familiar with other cloud storage applications, so keeping close to similar functionality is important in user understanding (Jakob’s Law). I found this to be true while conducting user testing. Most of the people reviewed the functionality by comparing it to the other cloud storage applications they use.

Providing too many options at the same time complicates the usability. This is the reason I decided to simplify the navigation on the mobile Interface (Hick’s Law).

I would have liked to have done more research into how people use cloud storage differently between work and home.
